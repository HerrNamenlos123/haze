# nodiscard Refactor: Type Modifier Implementation Plan

## Goal

Move `nodiscard` from a special field on `TaggedUnionDatatypeDef` to a proper type modifier
on `TypeUse` (like `inline`). It must be stackable, parseable like any other modifier, applicable
to any type, and have full meta-field support.

**Key invariant**: This mirrors how `inline`/`inlineByDefault` works. Types (structs and tagged
unions) can carry `nodiscardByDefault: true`, causing `makeTypeUse` to automatically set
`nodiscard = true` on any TypeUse created from them — unless overridden with `"force-no-nodiscard"`.

## How to run the compiler

```
bun run dev build
```

```typescript
export function makeTypeUse(
  sr: Semantic.Context,
  typeId: Semantic.TypeDefId,
  mutability: EDatatypeMutability,
  inline: boolean | "force-no-inline",
  nodiscard: boolean | "force-no-nodiscard",
  sourceloc: SourceLoc
) {
  const type = sr.typeDefNodes.get(typeId);

  // Compute shouldBeNodiscard
  const defNodiscardByDefault =
    (type.variant === Semantic.ENode.StructDatatype ||
      type.variant === Semantic.ENode.TaggedUnionDatatype)
      ? type.nodiscardByDefault
      : false;
  let shouldBeNodiscard: boolean;
  if (nodiscard === "force-no-nodiscard") {
    shouldBeNodiscard = false;
  } else {
    shouldBeNodiscard = defNodiscardByDefault || nodiscard;
  }

  if (type.variant === Semantic.ENode.StructDatatype) {
    let shouldBeInline = type.inlineByDefault;
    if (inline === "force-no-inline") {
      shouldBeInline = false;
    } else {
      shouldBeInline ||= inline;
    }

    for (const id of sr.typeInstanceCache) {
      const typeUse = sr.typeUseNodes.get(id);
      if (
        typeUse.mutability !== mutability ||
        typeUse.type !== typeId ||
        typeUse.inline !== shouldBeInline ||
        typeUse.nodiscard !== shouldBeNodiscard
      ) {
        continue;
      }
      return [typeUse, id] as const;
    }

    const instance = sr.b.addTypeInstance(sr, {
      mutability: mutability,
      inline: shouldBeInline,
      nodiscard: shouldBeNodiscard,
      type: typeId,
      sourceloc: sourceloc,
    });
    sr.typeInstanceCache.push(instance[1]);
    return instance;
  }

  // Non-struct path
  let shouldBeInline = inline;
  if (shouldBeInline === "force-no-inline") {
    shouldBeInline = false;
  }

  for (const id of sr.typeInstanceCache) {
    const typeUse = sr.typeUseNodes.get(id);
    if (
      typeUse.type !== typeId ||
      typeUse.mutability !== mutability ||
      typeUse.nodiscard !== shouldBeNodiscard
    ) {
      continue;
    }
    return [typeUse, id] as const;
  }

  const instance = sr.b.addTypeInstance<Semantic.TypeUse>(sr, {
    mutability: mutability,
    inline: shouldBeInline as boolean,
    nodiscard: shouldBeNodiscard,
    type: typeId,
    sourceloc: sourceloc,
  });
  sr.typeInstanceCache.push(instance[1]);
  return instance;
}
```

## Step 12 — Stdlib update: named `nodiscard` types

If stdlib defines `nodiscard union { ... }` (e.g., `Result`, `Option`), these will now be parsed
as `nodiscard` applied to the union TypeUse — meaning the TypeUse is nodiscard, not the TypeDef.
To restore the "by default nodiscard" behavior, the union must be named and defined with
`nodiscardByDefault: true` on the TypeDef.

For **anonymous** inline tagged unions like `nodiscard union { Ok: T, Err: E }`: the TypeUse
created will have `nodiscard: true`, which is correct behavior.

For named types (e.g., `type Result = nodiscard union { ... }`): the TypeAlias will resolve to
a TypeUse with `nodiscard: true`. Any use of `Result` as a type will go through alias resolution.
Check that alias resolution preserves the `nodiscard` flag on the TypeUse.
